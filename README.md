# namefreq

`namefreq` — консольная утилита на Go для подсчета частоты имен во входном текстовом файле.

Входной файл должен содержать одно имя на каждой строке. 
Пустые строки игнорируются, пробелы вокруг имени обрезаются.
Unicode-имена обрабатываются как обычные строки Go, поэтому имена вроде `Алёна`, `Миша`, `Дима` поддерживаются без дополнительной настройки.

## Задача

Требуется посчитать, сколько раз встретилось каждое имя.

Для этого я реализовала external aggregation:

- потоковое чтение входного файла;
- локальную агрегацию ограниченного размера;
- запись временных sorted chunk-файлов;
- k-way merge отсортированных chunk-файлов;
- вывод результата в выбранном порядке сортировки;
- очистку временных файлов после успешного выполнения и при ошибке.

## Установка и запуск

Запуск из корня проекта:

```bash
go run ./cmd/namefreq input.txt
```

Сборка бинарного файла:

```bash
go build -o bin/namefreq ./cmd/namefreq
```

Запуск собранной утилиты:

```bash
./bin/namefreq input.txt
```

## Флаги

```bash
namefreq [--sort=name|freq] [--chunk-size=128MB] [--tmp-dir=/tmp/namefreq] input.txt
```

Флаги:

- `--sort=name` — вывести результат, отсортированный по имени. Это режим по умолчанию.
- `--sort=freq` — вывести результат по убыванию частоты; при равной частоте используется сортировка по имени.
- `--chunk-size=128MB` — максимальный объем данных, который локальная агрегация удерживает до сброса chunk-файла.
- `--tmp-dir=/tmp/namefreq` — директория для временных chunk-файлов.

Поддерживаемые суффиксы размера: `B`, `KB`, `MB`, `GB`. 
Замечание: Если суффикс не указан, значение считается количеством байт.

## Формат вывода

Результат пишется в `stdout` в формате:

```text
name<TAB>count
```

Пример:

```text
Алёна	2
Дима	1
Миша	3
```

## Алгоритм

1. Утилита открывает входной файл и читает его построчно через потоковый reader.
2. Каждая строка нормализуется: удаляются пробелы по краям, пустые строки пропускаются.
3. Имена агрегируются в локальную `map[string]int64` до достижения `chunk-size`.
4. Готовый chunk записывается во временный файл через `os.CreateTemp`.
5. Записи внутри каждого chunk-файла сортируются по имени и сохраняются в формате `name<TAB>count`.
6. После обработки входного файла выполняется k-way merge всех отсортированных chunk-файлов через min-heap.
7. При совпадении имен во время merge счетчики суммируются.
8. Для `--sort=name` результат merge выводится напрямую.
9. Для `--sort=freq` финальные пары собираются в память, сортируются по частоте и выводятся.
10. Временные chunk-файлы удаляются после завершения pipeline.

## Почему решение подходит для больших файлов

Входной файл не загружается целиком в память. Обработка выполняется потоково, строка за строкой.

Количество уникальных имен в одном локальном chunk ограничивается параметром `--chunk-size`. После достижения лимита данные сбрасываются во временный отсортированный файл, а локальная `map` очищается.

На этапе merge одновременно хранится только одна текущая запись из каждого chunk-файла и небольшая heap-структура. Это позволяет объединять много отсортированных временных файлов без загрузки их содержимого целиком.

Ограничение режима `--sort=freq`: для сортировки по частоте необходимо собрать финальный список уникальных имен после merge. Это ожидаемый компромисс, потому что глобальный порядок по частоте нельзя вывести потоково из данных, отсортированных по имени.

## Назначение пакетов

- `cmd/namefreq` — CLI entrypoint.
- `internal/config` — конфигурация, флаги и валидация значений.
- `internal/counter` — потоковое чтение строк.
- `internal/chunk` — локальная агрегация, временные chunk-файлы и формат записей.
- `internal/merge` — k-way merge отсортированных chunk-файлов.
- `internal/output` — форматирование результата и режимы сортировки.
- `internal/app` — orchestration полного pipeline.
